================================================================================
SHARED VARIABLES
================================================================================

Both the EV model and the Avellaneda-Stoikov model are quoted off the same
upstream estimates, computed once in app/strategies/shared/ and passed into
both. Model-specific math (reservation price, spread, optimization) lives in
each model's own folder — see ev_model/math.txt and avellaneda_stoikov/math.txt.

Fair Value (FV) — adaptive, chosen per query:
    Kalman FV (preferred):
        A constant-velocity Kalman filter over the full transaction history.
        State = [price_level, velocity]. Each observation refines both
        jointly — irregular time gaps are handled by the state transition
        matrix, and observation noise keeps a single outlier sale from
        swinging the estimate the way it would a plain average. Used
        whenever there are >= 5 trades in the trailing 90-day window.
    Recency-weighted FV (fallback if not enough transactions):
        FV = (∑ w_i * x_i) / ∑ w_i,  where  w_i = e^(-λ * t_i)
        Used when recent data is too sparse to trust a velocity estimate.
        Recent trades carry more weight than old ones.

Volatility (σ):
    σ = sqrt( (∑ w_i * (x_i - FV)²) / ∑ w_i )
    Recency-weighted standard deviation around FV. Measures price
    uncertainty, and serves as the unit of distance for fill probability
    in both models.

Liquidity (κ):
    κ = N / W   (trades per day)
    MLE estimate of the Poisson order-arrival rate from transaction
    history: N = trade count, W = observation window in days. A higher κ
    means a more actively traded item.

Aggressiveness (a):
    a = max(1.0, sqrt(κ))
    Derived from liquidity and shared by both models so fill probability
    behaves consistently regardless of which model wins. Liquid items
    tolerate tighter spreads; illiquid items floor at a = 1.

Fill probability — shared functional form:
    P(fill | quote) = e^(-a * |quote - FV| / σ)

Expected value per side — shared functional form:
    EV_bid = P(fill | bid) * (FV - bid)
    EV_ask = P(fill | ask) * (ask - FV)

Inventory (q): net units currently held (positive = long, negative = short).
Each model interprets q differently when shaping its reservation price —
see the model-specific docs.


================================================================================
MODEL OVERVIEW
================================================================================

EV MODEL — simulate
    Heuristic. No closed-form spread. Searches candidate bid and ask
    multipliers independently and keeps whichever combination maximizes
    total expected value. See ev_model/math.txt.

AVELLANEDA-STOIKOV MODEL — derive
    Analytical. Solves a stochastic control problem (HJB equation) for the
    optimal reservation price and spread in closed form, then sweeps risk
    aversion and time horizon to pick the best-scoring combination. See
    avellaneda_stoikov/math.txt.

Both models run on every quote; the one with the higher total EV is passed
downstream to the decision engine (see app/decision/).
